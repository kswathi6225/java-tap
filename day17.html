Class & Object (Core Concept)
📌 Class = Blueprint
📌 Object = Real Entity

🚗 Example: Car Class

class Car {
    String name;
    float cost;
    float mileage;

    public void start() {
        System.out.println("Car is starting");
    }

    public void accelerate() {
        System.out.println("Car is accelerating");
    }

    public void stop() {
        System.out.println("Car is stopping");
    }
}

💡 Explanation:

  • Variables → State (Properties)
  • name, cost, mileage
  • 👉 These are called Instance Variables
  • Methods → Behavior
  • start(), accelerate(), stop()
🚀
Object Creation 
Car c = new Car();
Part Meaning
new Car() Object created in Heap
c Reference variable
= Assignment
🔥
Accessing Data 
System.out.println(c.name);

Type Default

Assigning Values

🔥
Calling Methods 
c.start();
c.accelerate();
c.stop();
❌ Wrong Way:
System.out.println(c.start()); // ERROR

💡 Reason:

  • 👉 start() → returns void
  • 👉 println() needs a value
🚀
Memory Representation (VERY IMPORTANT)

📌 Two Memory Areas:

  • ✔ Stack – main method, reference variables
  • ✔ Heap – objects

💡 Flow:

  • Main method → Stack
  • Object → Heap
  • Reference → Points to object
  • Method call → New stack frame
  • After execution → removed
🔥
Multiple Objects Concept 
Car c1 = new Car();
Car c2 = new Car();
💡 Important:
👉 Each object is independent 
c1.name = "BMW";
System.out.println(c2.name);
🧠 Output: null

❗ Why?

  • 👉 Different memory locations
  • 👉 No connection between objects
🚀
Key Concept
  • ✔ Same class → Multiple objects
  • ✔ Each object → Separate data

Static vs Non-Static (INTRO)
❌ Problem:
add(); // error

💡 Reason:

  • 👉 main() → static
  • 👉 add() → non-static
  • 👉 Static cannot access non-static directly

✔ Solution 1: Make Method Static

public static int add()

✔ Solution 2: Use Object

Day17 d = new Day17();
d.add();
🔥
Return Type Usage

✔ Storing Value

int result = d.add();
System.out.println(result);

✔ Direct Use

System.out.println(d.add());
❗ Important:
👉 Works only if return type is NOT void

Naming Conventions (VERY IMPORTANT)

🟣 Class → Pascal Case

  • ClassName
  • CarModel

🟢 Method & Variables → Camel Case

  • methodName()
  • calculateSum()
❌ Bad Practice:
classname | METHODNAME
