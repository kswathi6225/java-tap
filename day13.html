What is a Variable?
Definition:
A variable is a container used to store data

🧠 Real-Life Example:

  • Gold ring = Data
  • Box/locker = Variable

Example:

int a = 10;
a → variable (container)
10 → data
🔥
Types of Variables in Java
There are mainly 2 types:
✅ 1. Instance Variable
✅ 2. Local Variable
🚀
Instance Variables
Definition:
Variables declared directly inside a class (outside methods)

📌 Example:

class Demo {
    int a;
    float b;
    boolean c;
}

🔥 Key Points:

  • Declared inside class
  • Memory → Heap (inside object)
  • Always have default values
🎯
Default Values
Data Type Default Value
int 0
float 0.0
boolean false
char empty character
🔥
Object Creation & Access

Code:

class Demo {
    int a;
    float b;
    boolean c;
}

public class Main {
    public static void main(String[] args) {
        Demo d = new Demo();

        System.out.println(d.a);
        System.out.println(d.b);
        System.out.println(d.c);
    }
}
Output:
0
0.0
false

🧠 Why?

  • Object created in Heap
  • Variables automatically initialized by Java
🔥
Modifying Instance Variables

Code:

d.a = 100;
d.b = 45.5f;
d.c = true;

System.out.println(d.a);   // 100
System.out.println(d.b);   // 45.5
System.out.println(d.c);   // true
🚀
Local Variables
Definition:
Variables declared inside a method

📌 Example:

public class Main {
    public static void main(String[] args) {
        int a;
        float b;
        boolean c;
    }
}

❌ ERROR CASE:

System.out.println(a);

// Error: variable a might not have been initialized

🔥 Key Points:

  • Declared inside method
  • Memory → Stack
  • ❌ No default values
✅
Correct Way

Code:

int a = 99;
float b = 45.5f;
boolean c = true;

System.out.println(a);   // 99
System.out.println(b);   // 45.5
System.out.println(c);   // true

Another Way (Important):

int a;
a = 99;  // Declaration + assignment separately

System.out.println(a); // ✓ Valid (assigned before use)
⚠️ But: Cannot use before assignment!
🔥
Memory Execution (VERY IMPORTANT 🔥)

🧠 Program Flow:

  • Code written → RAM
  • Saved → Hard Disk
  • Run → loaded back to RAM
  • Executes inside → JRE
📦 JRE Memory Structure:
4 Segments:
1. Code Segment
2. Stack Segment
3. Heap Segment
4. Static Segment
🚀
Heap vs Stack

🟦 Heap:

  • Stores Objects
  • Stores Instance Variables

🟨 Stack:

  • Stores Local Variables
  • Stores Reference Variables

🔥 Example Flow:

Demo d = new Demo();

Step-by-step:
new Demo() → Object created in Heap
Address (say 1000) generated
d (reference) stored in Stack
d → points to object
🔥
Important Difference
Feature Instance Variable Local Variable
Location Inside class Inside method
Memory Heap Stack
Default Value ✅ Yes ❌ No
Initialization Automatic Must by programmer
🔥
Compilation Error (Important)

Happens when:

int a;
System.out.println(a);

✔ Error occurs at compile time
