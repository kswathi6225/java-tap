Rule 3 — Every Object Has Two Parts
👉 Every object consists of:
1️⃣ State (Properties) → "HAS"
2️⃣ Behavior → "DOES"
🚗 Example: Car Object
🔹 State (HAS)
  • Name
  • Cost
  • Mileage
  • Color
  • Seats
🔸 Behavior (DOES)
  • Start
  • Stop
  • Accelerate
  • Change gear
Object Concept In Java Code
State (HAS) Data Types (String, int, float...)
Behavior (DOES) Methods / Functions
🧠 Rule 3 Summary:
Every object HAS state (properties) → coded using data types
Every object DOES behavior → coded using methods

Basic Java Class Structure
👉 In Java: Everything must be written inside a class 
class Car {
    // State (Properties) — Data Types
    String name;
    double cost;
    float  mileage;

    // Behavior — Methods
    void start()  { /* ... */ }
    void stop()   { /* ... */ }
}

Important Interview Questions
Q1: What is Object-Oriented Programming?
OOP is a programming paradigm that views everything as objects, where each object has state (properties) and behavior (methods), and belongs to a class.
Q2: What is an object?
An object is a real, tangible instance of a class. It has state (data) and behavior (functions). Example: a specific car like "i20" is an object.
Q3: What is a class?
A class is a blueprint / template (imaginary) that defines the structure and behavior for objects. Example: "Car" is a class.
Q4: Difference between class and object?
A class is imaginary (blueprint), an object is real (instance). Class is created first, then objects are created from it.
Q5: What are state and behavior?
State = properties an object HAS (coded as data types). Behavior = actions an object DOES (coded as methods/functions).
Q6: Real-world example of OOP?
A Car: Class = "Car" (blueprint). Object = "i20". State = name, cost, mileage. Behavior = start(), stop(), accelerate().
Q7: Why is Java object-oriented?
Because in Java, everything must be written inside a class. Every program models the world as objects with state and behavior.
