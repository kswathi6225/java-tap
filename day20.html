Day 20

Jagged Arrays ⛓️

Arrays with unequal rows – memory efficiency & real-world data structures

Jagged Array
2D
3D
Memory Efficient
🔥
Quick Revision (From Day 19)

✔ Arrays = objects

  • Store homogeneous data
  • Types: 1D, 2D, 3D
  • Regular array → same number of columns
  • Rectangular structure
🚀
New Topic: Jagged Array
Definition:
A Jagged Array is an array where each row has a different number of columns

Example:

  • Class 1 → 3 students
  • Class 2 → 5 students
  • 👉 Not equal → Jagged Array
🔥
Why Jagged Array is Needed?
❌ Problem with Regular Array
int[][] a = new int[2][5];
If class has only 3 students: Remaining 2 → wasted memory

⚠️ Memory Waste Calculation:

  • 👉 Integer = 4 bytes
  • 47 unused elements → 47 × 4 = memory waste
✅ Solution: Use Jagged Array
Allocate only required memory ✔
Efficient ✔
Real-world use ✔
🚀
How to Create 2D Jagged Array

Step 1: Declare

int[][] a = new int[2][];

👉 Rows known
👉 Columns unknown

Step 2: Assign Each Row

a[0] = new int[3];
a[1] = new int[5];
💡 Key Idea: 2D Jagged Array = Array of 1D arrays
🔥
Internal Working (VERY IMPORTANT)
  • ✔ First → 2 rows created
  • ✔ Then → each row points to different arrays
👉 a[0] → size 3
👉 a[1] → size 5
🚀
Traversing Jagged Array

Input & Output:

for(int i = 0; i < a.length; i++) {
    for(int j = 0; j < a[i].length; j++) {
        System.out.print(a[i][j] + " ");
    }
    System.out.println();
}
💡 Important:
👉 Use: a.length for rows
👉 Use: a[i].length for columns
🔥
KEY DIFFERENCE (INTERVIEW)
Feature Regular Array Jagged Array
Shape Rectangular Irregular
Memory Wasted Efficient
Columns Same Different
🚀
Real World Thinking
👉 Jagged array = Real life data

Examples:

  • Classrooms with different students
  • Months with different days
  • Data tables with variable sizes
🔥
3D Jagged Array (ADVANCED)

💡 Problem Understanding:

  • 👉 2 Schools
  • School 1 → 3 classes
  • School 2 → 2 classes
  • Each class → different students
🚀
Creation Logic (STEP-BY-STEP)

Step 1: Create 3D Array

int[][][] a = new int[2][][];

Step 2: Define Classrooms

a[0] = new int[3][];
a[1] = new int[2][];

Step 3: Define Students

a[0][0] = new int[2];
a[0][1] = new int[3];
a[0][2] = new int[3];

a[1][0] = new int[2];
a[1][1] = new int[3];
💡 Concept Flow: 3D → 2D → 1D
🔥
Most Common Mistake ❌
❌ Directly assigning students:
a[0][0] = new int[2];
Without defining: a[0] = new int[3][];

✔ Always follow order:

  • 1️⃣ Blocks
  • 2️⃣ Rows
  • 3️⃣ Columns
🚀
Traversing 3D Jagged Array 
for(int i = 0; i < a.length; i++) {
    for(int j = 0; j < a[i].length; j++) {
        for(int k = 0; k < a[i][j].length; k++) {
            System.out.print(a[i][j][k] + " ");
        }
        System.out.println();
    }
    System.out.println();
}
🔥
Important Insight
👉 Creation is DIFFERENT
👉 Traversal is SAME

✔ Same loops work for:

  • Regular array
  • Jagged array
🚀
Execution Logic
👉 Flow:
1️⃣ i → block
2️⃣ j → row
3️⃣ k → column
