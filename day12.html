Integer Division (VERY IMPORTANT)

Code:

int a = 25;
int b = 2;
int c = a / b;
System.out.println(c);
Output: 12

🧠 Why?

  • 25 / 2 = 12.5
  • int cannot store decimals
  • Decimal part (0.5) is removed → truncation
🔥 KEY RULE: If both operands are int, result is always int
🚀
Getting Decimal Output

❌ Wrong Way:

float c = a / b;

// Still output = 12.0

❗ WHY?

  • a = int
  • b = int
  • Operation happens first → result = int (12)
  • Then stored in float → becomes 12.0

✅ Correct Way:

float c = (float)a / b;

// Output: 12.5
🔥 RULE: If any one operand is float → result is float
🎯
Data Type Priority (IMPORTANT)
Type Size
byte 1 byte
short 2 bytes
int 4 bytes
float 4 bytes
double 8 bytes

👉 Rule: Result always becomes larger data type

  • int + float → float
  • int + int → int
🔥
Implicit Type Casting

Code:

float a = 25;
👉 No error!

Why?

  • int → float (small → big)
  • Automatic conversion
Called:
Implicit Type Casting
❌
Why This Gives Error?

Code:

float a = 25.0;
❌ Error!

🧠 Reason:

  • 25.0 → double (default)
  • double → float (big → small)
  • Not allowed automatically

✅ Fix:

float a = 25.0f;
🔥
Explicit Type Casting

Code:

int c = (int)(a / b);

👉 Required when: Large data type → small data type

🔥
Arithmetic Operator Rule (VERY IMPORTANT)
👉 When using arithmetic operators: All smaller types (byte, short) → converted to int

❗ Example:

byte x = 5;
x = x * 10;  // ❌ ERROR

🧠 Why?

  • x → byte
  • 10 → int
  • Result → int
  • int → byte ❌ not allowed

✅ Fix:

x = (byte)(x * 10);
🔥
Another Example

Code:

byte b = 6;
b = b + 8;  // ❌ ERROR

🧠 Reason:

  • b + 8 → int
  • Cannot store in byte

✅ Fix:

b = (byte)(b + 8);
🔥
Octal Numbers (TRICKY 🔥)

Code:

int y = 010;
System.out.println(y);
Output: 8

🧠 Why? Numbers starting with 0 → octal (base 8)

010 = 8 (decimal)

Format Type
10 decimal
010 octal
0b10 binary
0x10 hexadecimal
⚠️ VERY IMPORTANT: Know all number systems!
🔥
Character to Integer

Code:

int ch = 'A';
System.out.println(ch);
Output: 65

🧠 Why? Java stores characters as ASCII/Unicode values

🔥
Pre vs Post Increment (REVISION)

Pre Increment:

int ch = 'A'; // 65
System.out.println(++ch); // 66
System.out.println(ch);   // 66
  • First increase
  • Then use

Post Increment:

System.out.println(ch++); // 65
System.out.println(ch);   // 66
  • First use
  • Then increase
🔥
Case Sensitivity

Code:

int c = 'A';  // 65
int C = 'a';  // 97
👉 Java is: Case Sensitive
🔥
Expression Example (IMPORTANT)

Code:

System.out.println(c++ + ++C);

👉 Step:

  • c++ → 65 (then becomes 66)
  • ++C → 98
Output: 163
🔥
Golden Rules (INTERVIEW)
  • Integer ÷ Integer → Integer
  • Decimal removed → truncation
  • One float → result float
  • Smaller → larger → implicit
  • Larger → smaller → explicit
  • Arithmetic → result at least int
  • Octal starts with 0
  • Char → ASCII value
  • Always solve step-by-step
