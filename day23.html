Problem with .equals() Method 
String s1 = "Alex";
String s2 = "Charlie";
s1.equals(s2);   // false

👉 Output: false
❗ Problem:
It only tells → equal or not
❌ Does NOT tell:
• Which is bigger?
• Which comes first?
🔥
compareTo() Method (MOST IMPORTANT 🔥)
Purpose:
Used to compare strings AND also determine order

✅ Syntax

s1.compareTo(s2);

🎯 Return Type:
👉 Returns integer (NOT boolean) 

Case        Output
s1 > s2     Positive
s1 < s2     Negative
s1 == s2    0
🚀
How compareTo() Works

👉 Compares character by character

Example:
String s1 = "Sachin";
String s2 = "Saur";

Comparison:
S == S ✔
a == a ✔
c vs u ❌

👉 ASCII:
c = 99
u = 117

👉 Result:
99 - 117 = -18

✔ Output = Negative → s1 < s2
🔥
Cases of compareTo()

✅ Case 1: s1 < s2

"abc".compareTo("abd");  // negative

✅ Case 2: s1 > s2

"zebra".compareTo("apple");  // positive

✅ Case 3: Equal Strings

"java".compareTo("java");  // 0
🚀
Special Case (Length Difference) 
"java".compareTo("javac");  // negative

👉 Why?
First 4 chars same
Second string has extra character

✔ Shorter string = smaller
🔥
Difference: equals() vs compareTo()
Feature equals() compareTo()
Return type boolean int
Output true/false + / - / 0
Order info ❌ No ✅ Yes
Interview importance High VERY HIGH 🔥
🚀
Using compareTo() in Code 
int result = s1.compareTo(s2);

if(result > 0) {
    System.out.println("s1 is greater");
} else if(result < 0) {
    System.out.println("s1 is smaller");
} else {
    System.out.println("Both are equal");
}
🔥
Important String Methods

✅ 1. length()

s.length();
  • Returns number of characters
⚠️ Note: In arrays → length (variable) | In strings → length() (method)

✅ 2. charAt(index)

s.charAt(0);
  • Returns character
  • Index starts from 0

✅ 3. toLowerCase() / toUpperCase()

s.toLowerCase();
s.toUpperCase();
⚠️ Important: Does NOT modify original string. Creates new string (immutable concept)

✅ 4. compareToIgnoreCase()

s1.compareToIgnoreCase(s2);
  • Ignores uppercase/lowercase

✅ 5. indexOf()

s.indexOf('a');
  • Returns first occurrence index

✅ 6. lastIndexOf()

s.lastIndexOf('a');
  • Returns last occurrence

✅ 7. split()

String[] arr = s.split(" ");

Example:
"I love Java" → ["I", "love", "Java"]
  • Converts string → array

✅ 8. startsWith() / endsWith()

s.startsWith("Ja");
s.endsWith("va");
  • Returns boolean

✅ 9. toCharArray()

char[] arr = s.toCharArray();
  • Converts string → char array

✅ 10. substring()

🔹 1 Parameter

s.substring(5);

✔ From index → end

🔹 2 Parameters

s.substring(2, 6);

✔ Start inclusive
✔ End exclusive

✅ 11. replace() / replaceAll()

s.replace('a', 'x');

✔ Replaces characters
🔥
Important Concept (IMMUTABILITY) 
String s = "Java";
s.toLowerCase();
System.out.println(s);

👉 Output: Java

✔ WHY?
String is immutable
New object created
Original unchanged
🚀
concat() Memory Concept 
String s1 = "Tap";
s1.concat("Academy");

👉 New object created in heap
👉 Original NOT modified

✔ Garbage collector removes unused objects

Interview Important Points
  • ✔ compareTo() → returns int
  • ✔ equals() → returns boolean
  • ✔ Strings are immutable
  • ✔ Methods create new objects
  • ✔ length() vs length difference
  • ✔ charAt returns char
