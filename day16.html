    // method body

Return Type What method returns
Method Name Name of method

Important Terminology (NEW)

Parameters vs Arguments

Parameters
  • Variables in method definition
  • public void add(int a, int b)
  • a, b → Parameters
Arguments
  • Values passed during method call
  • add(10, 20);
  • 10, 20 → Arguments
KEY LINE: Parameters = Receiving | Arguments = Sending
🚀
Type 3 Method – Input + No Output

Example:

class Calculator {
    int c;

    public void add(int a, int b) {
        c = a + b;
        System.out.println(c);
    }
}

Calling:

Calculator cal = new Calculator();

int num1 = 50;
int num2 = 40;

cal.add(num1, num2);
Output: 90

Key Points:

  • Takes input via parameters
  • Does NOT return → uses void
  • Prints directly
🔥
Memory Execution (VERY IMPORTANT)

Flow:

  • Main method → Stack frame created
  • Object → Heap memory
  • Local variables (num1, num2) → Stack
  • Method call → New stack frame
  • Parameters (a, b) get values
  • Execution → prints result
  • Stack frame removed
Important:
✔ Local variables → Stack
✔ Instance variables → Heap
✔ Stack → LIFO (Last In First Out)
🚀
Type 4 Method – Input + Output (MOST IMPORTANT)

Example:

class Calculator {
    int c;

    public int add(int a, int b) {
        c = a + b;
        return c;
    }
}

Calling:

Calculator cal = new Calculator();

int num1 = 50;
int num2 = 40;

int result = cal.add(num1, num2);
System.out.println(result);
Output: 90

Key Points:

  • Takes input
  • Returns value
  • MOST reusable method
🔥
Return Concept (IMPORTANT)

What does return do?

  • Sends:
  • Value
  • Control
  • back to the caller

Flow:

  • Method executes
  • return sends value
  • Control goes back to main
  • Value stored in variable
🚀
Pass by Value (VERY IMPORTANT)

Concept:

cal.add(num1, num2);

👉 Values of num1, num2 are copied
KEY IDEA: Changes inside method DO NOT affect original variables

Example:

int a = 10;
// Passed to method → copy created
🔥
Local vs Instance Variables
Type Where Stored
Local Inside method Stack
Instance Inside class Heap
🚀
Default Values
  • Instance variables → default value given
  • Local variables → NO default (must initialize)
🔥
Code Reusability (MAIN PURPOSE)
❌ Without Method:
Repeat same code multiple times
✅ With Method:
Write once → Call multiple times
🚀
Example – Pattern Program 
public void printSquare(int n) {
    for(int i = 0; i < n; i++) {
        for(int j = 0; j < n; j++) {
            System.out.print("* ");
        }
        System.out.println();
    }
}

Calling:

printSquare(5);
printSquare(3);
printSquare(8);
Advantage:
✔ One method → multiple outputs
✔ Reduced code
🔥
Taking Input (Scanner) 
Scanner sc = new Scanner(System.in);
int n = sc.nextInt();

printSquare(n);
  • Input taken in main method
  • Passed to method
🚀
Method Execution Summary
  • ✔ Execution starts from main()
  • ✔ Stack frame created
  • ✔ Object → Heap
  • ✔ Method call → New stack frame
  • ✔ Return → control back
  • ✔ Garbage collector cleans heap
