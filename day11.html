Increment & Decrement ➕

Pre & Post increment/decrement operators in Java

Operators
Pre/Post
Expressions
Overflow
🔥
What is Increment & Decrement?

👉 Increment (++)

Increases value by 1

👉 Decrement (--)

Decreases value by 1

🧠 Basic Example:

int a = 5;

a++;   // increment
a--;   // decrement
🚀
Why Do We Use It?

Instead of writing:

a = a + 1;

We write:

a++;
✔ Short | ✔ Cleaner | ✔ Industry standard
🔥
Increment Operator

Syntax: a++

Means:
a = a + 1

✅ Example:

int a = 5;
a++;
System.out.println(a); // 6
🔥
Decrement Operator

Syntax: a--

Means:
a = a - 1

✅ Example:

int a = 5;
a--;
System.out.println(a); // 4
⚡
Types of Increment/Decrement

There are 2 types:

  • ✅ Pre Increment / Decrement (++a / --a)
  • ✅ Post Increment / Decrement (a++ / a--)
🚀
PRE INCREMENT (++a)
Definition:
First increase → then use

🧠 Example:

int a = 5;
int b = ++a;

System.out.println(a); // 6
System.out.println(b); // 6
🔥 Rule: Increase first → Then assign
🚀
POST INCREMENT (a++)
Definition:
First use → then increase

🧠 Example:

int a = 5;
int b = a++;

System.out.println(a); // 6
System.out.println(b); // 5
🔥 Rule: Use first → Then increase
🎯
IMPORTANT CONCEPT (VERY IMPORTANT)
Key Point:
Pre/Post DO NOT affect the original variable immediately in thinking. They affect where the value is used (like b).
"a value remains same logic, b changes based on pre/post"
🧠
Easy Trick (REMEMBER THIS)
Type Rule
++a First increase → then use
a++ First use → then increase
🔥
Example (VERY IMPORTANT)

Code:

int a = 5;
int b = ++a + ++a;

Step by step:

  • First ++a → 6
  • Second ++a → 7
  • b = 6 + 7 = 13
  • a = 7

Another Example:

int a = 5;
int b = a++ + a++;

Step by step:

  • First a++ → use 5 → then becomes 6
  • Second a++ → use 6 → then becomes 7
  • b = 5 + 6 = 11
  • a = 7

Mixed Example:

int a = 5;
int b = ++a + a++;

Step by step:
++a → 6
a++ → use 6 → then becomes 7

b = 6 + 6 = 12
a = 7
🔥
Complex Expression (INTERVIEW LEVEL)

Code:

int a = 5;
int b = a++ + ++a + a-- + --a + ++a + --a;
Final Answer:
✔ a = 5
✔ b = 35
⚠️ Solve step-by-step on paper always!
⚠️
VERY IMPORTANT RULE
👉 Always:
✔ Write step by step
✔ Don't solve in mind
✔ Track value after each operation
🔥
Byte Overflow (IMPORTANT)

Code:

byte b = 127;
b++;
b++;
System.out.println(b);
Output: -127

🧠 Why?

  • Byte range: -128 to 127
  • After 127: wraps to -128
  • Then: -127
Called:
Wrapping / Overflow
🚀
Important Concept
Initialization vs Execution:
Initialization: Error if out of range
Execution: Wrap around happens
🔀
Type Casting Connection

Code:

double a = 128;
byte b = (byte)a;

👉 Why explicit?

  • double = 8 bytes
  • byte = 1 byte
  • Data loss → explicit casting required
🎯
Interview Answer (PERFECT WAY)

❌ Wrong:

"Data loss happens"

✅ Correct:

"Double occupies 8 bytes and byte occupies 1 byte. Assigning a larger data type to a smaller one causes data loss, so explicit type casting (narrowing) is required."

🔥
Final Quick Revision
  • ++ → increment
  • -- → decrement
  • ++a → first increase
  • a++ → first use
  • Pre/Post affects result variable
  • Always solve step-by-step
  • Byte overflow → wrapping
  • Explicit casting → for big → small
