How Data is Stored in Memory (IMPORTANT)

Example: byte a = 24;

Step 1: Convert to Binary (Base 2)

Use divide by 2 method: 

24 ÷ 2 → remainder 0
12 ÷ 2 → remainder 0
6 ÷ 2 → remainder 0
3 ÷ 2 → remainder 1
1 ÷ 2 → remainder 1

Step 2: Read bottom to top

11000

Step 3: Make it 8 bits (byte = 8 bits)

00011000
✔ This is how 24 is stored in RAM!
📌 Key Point
Every number you use in Java is first converted to binary before being stored in RAM. Data types define how many bits are used.
⚡
MSB (Most Significant Bit)

👉 Definition: The first bit in binary representation 

00011000
↑
└─ MSB (Most Significant Bit)
MSB Value Meaning
0 Positive number
1 Negative number

Example:

00011000 → MSB = 0 → Positive ✓
11100111 → MSB = 1 → Negative ✓
So computer knows if a number is positive or negative just by looking at the first bit!
🔴
Negative Numbers (VERY IMPORTANT)

Example: byte a = -24;

Step 1: Convert +24 to binary

00011000

Step 2: 1's Complement (flip/invert all bits)

11100111

0 becomes 1, 1 becomes 0

Step 3: Add 1 (2's Complement)

11100111 + 1 = 11101000
✔ Final answer = 11101000 is binary of -24
📌 Key Rule
Positive numbers: Convert using Base 2
Negative numbers: Convert using Base 2 + 2's Complement
This is called 2's Complement representation - used by Java for storing negative numbers!
🧠
Important Concept
Why data types are called converters:
Real data (human-readable) → Binary (0 & 1) → Stored in RAM

Data types handle this complete conversion process!

RAM
✔ This is the REAL meaning of data types - not just "types of data"
🔢
Real Number Data Types

👉 Used for decimal values

Examples: 5.11, 6.25, 45.5

🟡 1. float

  • Size: 4 bytes (32 bits)
  • Range: ~ -3.4 × 10^38 to +3.4 × 10^38
  • Code: float a = 45.5f;
  • Note: Must add f suffix

🔵 2. double

  • Size: 8 bytes (64 bits)
  • Range: ~ -1.7 × 10^308 to +1.7 × 10^308
  • Code: double a = 45.5;
  • Default: All decimal numbers are treated as double by default
Both use special internal representation to store huge values in limited bits!
⚠️
VERY IMPORTANT RULE (INTERVIEW QUESTION)

👉 By default:

All decimal numbers → double

❌ Wrong Code:

float a = 45.5;   // ERROR

Error: cannot convert from double to float

🔥 Why?

  • double = 8 bytes
  • float = 4 bytes
  • Bigger container → smaller container ❌ Not allowed
  • Data loss risk!

✔ Correct Solution:

float a = 45.5f;   // ✓ With f suffix

The f suffix tells Java: "Treat this as float, not double"

Rule: Always use f suffix for float values!
🎯
Float vs Double (IMPORTANT)
Feature float double
Precision Low High
Bytes 4 8
Decimal Accuracy ~6 digits ~15+ digits
Default ❌ Not default ✓ Default for decimals

🧠 Examples:

65              → int (no decimal)
65.2            → could be float
65.283649521    → should be double (high precision)
Rule of Thumb: Need high precision? Use double. Need to save memory? Use float.
🚀
Why float & double can store large values?

👉 Because of:

  • Compression
  • Special internal representation
How it works:
float & double don't store numbers directly. They use a special format (IEEE 754) that efficiently encodes huge values using limited bits. Like scientific notation: 3.4 × 10^38
They trade exact precision for the ability to store enormous ranges!
🔤
Character Data (INTRO)

👉 Reality:

RAM cannot store: A, B, C, D...

RAM only stores: 0 and 1

🧠 Idea:

Assign binary codes to characters:

A → 00
B → 01
C → 10
D → 11
✔ This is how characters are stored!
Key Concept:
Each character is mapped to a unique binary code. This code is stored in RAM, then converted back to characters when displayed.
💡
Code Length Formula
Formula:
If symbols = N
Length = log₂(N)

Examples:

Total Symbols Code Length (bits)
4 2 bits (2² = 4)
8 3 bits (2³ = 8)
16 4 bits (2⁴ = 16)
256 8 bits (2⁸ = 256)
This formula helps us determine how many bits we need for any set of symbols!
🇺🇸
ASCII Table (VERY IMPORTANT)

📌 Full Form:

American Standard Code for Information Interchange

👉 Total Symbols:

  • 128 characters total
  • Bits required: 2⁷ = 128
  • So: 7-bit code system

📋 Includes:

  • Numbers: 0–9
  • Alphabets: A–Z, a–z
  • Special symbols: @, #, %, &, etc.
  • Control characters: space, newline, tab, etc.
Character ASCII Value Binary
A 65 01000001
a 97 01100001
0 48 00110000
@ 64 01000000
Every character in Java has an ASCII value! When you store 'A', Java actually stores 65 (its ASCII code).
🔥
Final Summary

✅ Integer Data

  • byte, short, int, long
  • Stored using binary
  • MSB indicates sign
  • Negative = 2's Complement

✅ Real Numbers

  • float, double
  • Default = double
  • Use f suffix for float
  • Store huge ranges efficiently

✅ Characters

  • Stored using ASCII codes
  • 128 total ASCII characters
  • 7-bit representation
  • Each character = binary number

👉 Be ready to answer:

  • What is data type? → Converter between real & binary data
  • How is data stored in RAM? → Converted to binary using data type
  • What is MSB? → First bit; 0=positive, 1=negative
  • What is 2's Complement? → Method to store negative numbers
  • Difference between float & double? → Bytes (4 vs 8), precision, default
  • Why does float need f suffix? → Because default is double (8 bytes)
  • What is ASCII? → Character encoding standard with 128 symbols
  • How are characters stored? → As ASCII codes (binary numbers)
