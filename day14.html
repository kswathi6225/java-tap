Pass by Value & Reference 🔗

Understanding how Java passes data to methods

Introduction

Today's Concepts:

  • Pass by Value
  • Pass by Reference
Very important for:
✔ Object-Oriented Programming
✔ Future topics (Collections, Objects, Methods)
🔥
What is Pass by Value?
Definition:
Passing a copy of the value from one variable to another

Example:

int a = 1000;
int b = a;

🧠 Explanation:

  • a stores → 1000
  • b gets → copy of 1000
Output:
System.out.println(a); // 1000
System.out.println(b); // 1000
🔥
Internal Working

Step-by-step:

  • a → 1000
  • b created (empty)
  • b = a → copy value
  • Both have independent values
🔥
Key Concept (IMPORTANT)

After copying:

b = 2000;

Output:
a = 1000
b = 2000
✔ Changing b does NOT affect a

Another Case:

a = 2000;

Output:
a = 2000
b = 1000
✔ Still independent
🚀
Why?

Because:

  • Only value is copied
  • No connection between variables
🔥
Real-Life Example (XEROX COPY)

📖 Xerox Copy Analogy:

  • Original book → a
  • Xerox copy → b
✔ If you write in original → copy doesn't change
✔ If you tear copy → original doesn't change

This is Pass by Value
🔥
Key Points
  • Copy is created
  • No link between variables
  • Changes are independent
🚀
Pass by Reference
Definition:
Passing reference (address) instead of value
💡 In Java: We say reference, not address
🔥
Important Concept Before That

Java is:

Object-Oriented Programming Language

Rules:
✔ World is collection of objects
✔ Every object belongs to a class
✔ Every object has:
  - State (variables)
  - Behavior (methods)
🔥
Example Using Class

Step 1: Create Class

class Car {
    String name;
    int seats;
    float cost;
}

Step 2: Main Class

public class Demo {
    public static void main(String[] args) {
        Car a = new Car();
    }
}
🔥
Memory Understanding

When object is created:

Car a = new Car();

🧠 What happens:

  • Object created in Heap
  • Address (e.g., 1000) generated
  • a stores address in Stack
🔥
Default Values
Variable Type Default
name String null
seats int 0
cost float 0.0
💡 Important: String is NOT a primitive → it is an object
🚀
Modifying Object Values 
a.name = "Maruti";
a.seats = 5;
a.cost = 8.66f;

Output:
System.out.println(a.name);   // Maruti
System.out.println(a.seats);  // 5
System.out.println(a.cost);   // 8.66
🔥
Pass by Reference Concept

Now imagine:

Car a = new Car();
Car b = a;

🧠 What happens:

  • a → points to object
  • b → also points to SAME object
🔥 Important Case: 
b.name = "BMW";

Output:
a.name = "BMW"
b.name = "BMW"

✔ Both changed!
🚀
Why?

Because:

  • Both a and b refer to same object
  • No copy created
🔥
Key Difference
Feature Pass by Value Pass by Reference
What is passed Copy Reference
Memory Separate Same
Change effect No impact Affects both
Used in Primitives Objects
🔥
Important Interview Point
👉 Java is:
Always Pass by Value
⚠️ BUT:
For objects → value = reference

👉 So it behaves like pass by reference
🔥
Golden Points
  • Primitive → Pass by Value
  • Object → Reference passed (value of address)
  • Changes reflect in objects
  • Very important for OOP
⚡ FINAL REVISION:

Remember:
int a = 10 → copy
Car a = new Car() → reference
