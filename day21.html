Day 21

Objects in Arrays 🎬

Storing objects in arrays – references, homogeneity, and practical applications

Objects
Arrays
References
Homogeneous
🔥
Introduction
👉 Till now:
Arrays stored primitive data (int, float, char, etc.)

👉 Today:
Can arrays store objects? ✅ YES
Understanding real-world usage + drawbacks
🚀
Objects in Array
Key Concept:
Array can store: Primitive types ✅ and Objects ✅
🔥
Example: Student Class 
class Student {
    String name;
    int id;
    String branch;
}

✔ Creating Objects

Student s1 = new Student();
Student s2 = new Student();
Student s3 = new Student();

✔ Assign Values

s1.name = "Siddhu";
s1.id = 1;
s1.branch = "CSE";

s2.name = "Shahat";
s2.id = 2;
s2.branch = "Mechanical";

s3.name = "Chalan";
s3.id = 3;
s3.branch = "Civil";
🚀
Storing Objects in Array

✔ Create Array

Student[] ar = new Student[3];

👉 Array stores Student type objects
⚠️ Default Value:
ar[0] → null
ar[1] → null
ar[2] → null
✔ Because: Objects default = null

✔ Store Objects

ar[0] = s1;
ar[1] = s2;
ar[2] = s3;

✔ Access Data

System.out.println(ar[0].name);
System.out.println(ar[0].id);
System.out.println(ar[0].branch);
🔥
Important Concept: Reference
👉 Array stores reference (address) of object
👉 Not actual object

💡 Example:

ar[0] = s1;

👉 Both point to same object
🔥 RESULT
ar[0].name = "Alex";
System.out.println(s1.name);
👉 Output → Alex
✔ Because: Pass by reference concept
🚀
Multiple References to Same Object 
ar[0] = s1;
ar[1] = s1;

👉 Both indices point to SAME object
✔ Any change affects both
🔥
Important Rule (VERY IMPORTANT)
❗ Arrays are Homogeneous
👉 Same type only allowed
❌ WRONG
Customer c = new Customer();
ar[3] = c; // ERROR

👉 Because:
Array = Student type
Object = Customer type
❌ Type mismatch error

✔ Arrays store:
👉 Only SAME TYPE objects

🚀
Real World Example

👉 E-commerce app:

  • Store customers in array
  • Access using index
  • customers[49] → 50th customer
🔥
Drawbacks of Arrays

❌ 1. Homogeneous Data

  • Only same type allowed

❌ 2. Fixed Size (Static)

int[] a = new int[5];

👉 Cannot increase/decrease

❌ 3. Array Index Out of Bounds

a[5] = 60;

👉 ERROR
👉 Exception: ArrayIndexOutOfBoundsException

❌ 4. Contiguous Memory Requirement

👉 Array needs: Continuous memory

💡 Problem:
RAM = fragmented
May not get continuous space

👉 Large arrays may fail
🚀
Array Declaration Types
✔ Valid
int[] a;
int a[];
❌ Invalid
[] int a; // WRONG

👉 Rule: Data type FIRST
🚀
Array Initialization (Shortcut)

✔ Instead of:

int[] a = new int[3];
a[0] = 10;
a[1] = 20;
a[2] = 30;

✔ Use:

int[] a = {10, 20, 30};
🔥
2D Array Initialization 
int[][] a = {
    {10, 20},
    {30, 40}
};

✔ Concept:
👉 2D array = multiple 1D arrays
🔥
Final Key Points
  • ✔ Arrays can store objects
  • ✔ Objects stored as references
  • ✔ Pass by reference concept
  • ✔ Arrays are homogeneous
  • ✔ Fixed size
  • ✔ Contiguous memory needed
  • ✔ Supports shortcut initialization
⚡ FINAL REVISION
👉 Objects in array = MOST IMPORTANT 🔥
👉 Pass by reference = INTERVIEW QUESTION
👉 Drawbacks = MUST REMEMBER
