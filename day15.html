What is a Method?
Definition:
A method is a block of code used to perform a specific task

Structure:

access_modifier return_type methodName() {
    // method body
}

🧠 Example:

public void add() {
    System.out.println("Hello");
}
🔥
Parts of a Method

✔ 1. Access Modifier

  • public, private, etc.
  • Controls visibility

✔ 2. Return Type

  • What method gives back
  • Example: int, void

✔ 3. Method Name

  • Should be meaningful
  • Example: add(), calculateSum()

✔ 4. Parentheses ()

  • Used to take input

✔ 5. Body { }

  • Contains logic
🔥
Why Methods?

📌 Problem without methods:

  • Code repeated again & again
  • Difficult to manage

📌 Solution: Methods provide:

  • Code Reusability
  • Clean code
  • Easy debugging
🚀
Types of Methods (MAIN CONCEPT)
💡 There are 4 types of methods:
🔹 1. No Input, No Output
🔹 2. No Input, With Output
🔹 3. With Input, No Output
🔹 4. With Input, With Output
🔥
Type 1: No Input, No Output

📌 Example:

class Calculator {
    int a = 50;
    int b = 40;

    public void add() {
        int c = a + b;
        System.out.println(c);
    }
}

📌 Calling Method:

Calculator c = new Calculator();
c.add();
🧠 Output: 90

💡 Key Point:

  • Prints result
  • Does NOT return anything
  • 👉 return type = void
🔥
Method Execution (IMPORTANT)

Steps:

  • Main method starts → stack frame created
  • Object created → heap memory
  • Method call → new stack frame
  • Execution happens
  • Method removed from stack
👉 Stack follows:
LIFO (Last In First Out)
🔥
Stack vs Heap

📌 Stack:

  • Stores methods & local variables
  • Works on LIFO

📌 Heap:

  • Stores objects
  • Contains instance variables
🚀
Garbage Object
Definition:
Object without reference

📌 Example:

Calculator c = new Calculator();

After execution:

c removed
Object remains → becomes garbage
👉 Cleaned by:
Garbage Collector (Automatic in Java)
🔥
Type 2: No Input, With Output

📌 Example:

class Calculator {
    int a = 50;
    int b = 40;

    public int add() {
        int c = a + b;
        return c;
    }
}
🔥
Calling This Method 
Calculator c = new Calculator();
int result = c.add();
System.out.println(result);
🧠 Output: 90
🟢
Important Difference: Printing vs Returning
Printing Returning
Shows output directly Sends value back
System.out.println() return
No reuse Reusable
🔥
Memory Flow (IMPORTANT)

👉 When method returns:

  • Value + control → goes back to main
  • Stored in variable
  • Then printed
🔥
Return Type Rules

📌 Rule: Return type must match returned value

❌ Wrong:

public void add() {
    return 10; // ERROR
}

✅ Correct:

public int add() {
    return 10;
}
🔥
Type Casting Connection (From Lecture)

📌 Example:

int a = 50;
float b = 40;

👉 a + b → result = float

❌ Wrong:

int c = a + b;

✅ Fix:

int c = (int)(a + b);
